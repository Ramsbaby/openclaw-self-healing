# Discord 채널별 실시간 품질 Self-Check

## 개요

각 채널의 systemPrompt에 응답 전 self-validation 로직을 추가하여 실시간으로 품질을 검증합니다.

## Self-Check 프로세스

### #jarvis (일반)
**응답 전 체크:**
- [ ] 2000자 이하인가?
- [ ] 도구 3회 이상 시 중간 보고 했는가?
- [ ] 링크 3개 이상 시 <> 감쌌는가?
- [ ] 실패 시 대안 제시했는가?

### #jarvis-market (시장)
**응답 전 체크 (TQQQ):**
- [ ] USD 가격 있는가?
- [ ] KRW 환산 있는가?
- [ ] 변동률 % 있는가?
- [ ] Stop-Loss 거리 계산했는가?
- [ ] 데이터 출처 명시했는가?
- [ ] 타임스탬프 있는가?
- [ ] 리스크 섹션 있는가?

**누락 시:**
- 즉시 자가 수정
- 또는 "⚠️ 데이터 불완전. X 누락. 수동 확인: [링크]"

### #jarvis-system (시스템)
**응답 전 체크:**
- [ ] 긴급도 이모지 있는가?
- [ ] 로그 10줄 이하인가?
- [ ] 민감 정보 마스킹했는가?
- [ ] 중복 이벤트 1시간 내인가?

### #jarvis-dev (개발)
**응답 전 체크:**
- [ ] 코드블록에 언어 명시했는가?
- [ ] 에러는 5단계 분석했는가?
- [ ] 성능 지표 포함했는가?
- [ ] ChatGPT 톤 사용 안 했는가?

## 구현

systemPrompt에 다음 추가:

```
**응답 전 Self-Check:**
[채널별 체크리스트]
누락 시 자가 수정 또는 경고 포함
```

## 로깅

Self-check 실패 시:
- `~/openclaw/memory/quality-warnings-YYYY-MM-DD.log`에 기록
- 주간 감사 리포트에 포함
